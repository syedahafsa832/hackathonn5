## Phase 0: Research — Resolved

See [research.md](./research.md) for full findings. Summary:

| Item | Key Finding | Resolution |
|------|-------------|------------|
| A — Auth | `sub` in v1 token = tenant_id; middleware looks in `users` table (wrong) | Fallback path: query `tenants` table if `users` lookup fails |
| B — Stats | `/admin/conversations` doesn't exist in hack5; silently returns [] | Use `/api/tickets` + `/api/v1/actions/stats` |
| C — Fields | `channel` may be null in DB; `customer_email` exists | Default `channel` to `"email"` in list transform |
| D — Worker | Main loop already has try/except | Harden inner per-brand loop; add sleep(5) on outer error |
| E — Email | `approve-ai` endpoint never calls `send_email` | Add Gmail send call in v2_tickets.py approve-ai handler |
| F — KB UI | Backend endpoints exist; no Settings tab | Add 3rd tab to Settings.jsx |
| G — Wizard | Onboarding.jsx complete; Dashboard doesn't redirect | Add `GET /api/brands` check on Dashboard mount |
| H — Threading | thread_id captured but not used for dedup | Query tickets by gmail_thread_id before creating new ticket |
| I — Notifications | `useNotifications` hook complete but not wired | Import and call in Dashboard.jsx |

## Phase 1: Design

### Item A — Auth Middleware Fix

**File**: `backend/src/api/middleware/auth_middleware.py`
**Also**: `backend/src/services/supabase_auth_service.py` (add `get_tenant_context` method)

**Design**:

```
get_current_user(credentials):
  1. verify_jwt(token) → payload (or raise 401)
  2. sub = payload["sub"]
  3. user_context = get_user_context(sub)  # tries users table
  4. if user_context is None:
       tenant = auth_service.get_tenant(sub)  # tries tenants table (v1 path)
       if tenant is None: raise 401
       user_context = UserContext(
         user_id=tenant["id"],
         supabase_auth_id=tenant["id"],
         organization_id=tenant["id"],
         email=tenant["email"],
         role=UserRole.ADMIN,
         brands=[]
       )
  5. return AuthenticatedContext(user=user_context)
```

`get_optional_auth` — same fallback pattern, returns None instead of raising.

**Acceptance**: `curl -H "Authorization: Bearer <v1_token>" /api/v1/actions/stats` → 200

### Item B — Dashboard Stats

**File**: `ai-ops-console/src/api/services.js`

**Design** — replace `getStats()`:
```javascript
getStats: async () => {
  const [ticketsRes, actionsRes] = await Promise.all([
    client.get('/api/tickets').catch(() => ({ data: [] })),
    client.get('/api/v1/actions/stats').catch(() => ({ data: {} })),
  ]);
  const tickets = Array.isArray(ticketsRes.data) ? ticketsRes.data : [];
  const a = actionsRes.data || {};
  const open = tickets.filter(t => t.status === 'open' || !t.status);
  const resolved = tickets.filter(t => t.status === 'resolved');
  const aiHandledPct = tickets.length > 0
    ? Math.round((resolved.length / tickets.length) * 100) : 0;
  return {
    activeConversations: open.length,
    totalConversations: tickets.length,
    escalatedChats: tickets.filter(t => t.status === 'escalated').length,
    investorLeads: a.pending_count ?? 0,
    studentLeads: a.total ?? 0,
    aiHandledPct,
    avgLatency: 0,
  };
}
```

**Acceptance**: Dashboard shows non-zero Active Conversations when tickets exist in DB.

### Item E — Email Send

**File**: `backend/src/api/routes/v2_tickets.py` — `approve_ai_response` handler (~line 448)

**Design** — add send logic inside the handler after updating ticket status:
```python
# After: supabase_update("tickets", ..., {"human_approved": True})
brand_id = ticket.get("brand_id") or ticket.get("store_id")
if brand_id:
    brands = supabase_select("brands", {"id": f"eq.{brand_id}", "gmail_connected": "is.true"})
    if brands:
        reply_body = ticket.get("ai_response") or ticket.get("ai_draft")
        subject = ticket.get("subject", "Support")
        send_result = await brand_gmail_service.send_email(
            brands[0], ticket["customer_email"],
            f"Re: {subject}", reply_body
        )
        if send_result.get("success"):
            supabase_update("tickets", {"id": f"eq.{ticket_id}"},
                {"status": "resolved", "email_sent": True})
            return {"success": True, "message": "AI response approved and sent"}
        else:
            return {"success": False, "error": send_result.get("error", "Send failed")}
    else:
        return {"success": False,
                "error": "No Gmail connected for this brand. Go to Brands → Connect Gmail first."}
```

**Acceptance**: Approve-ai on ticket with Gmail connected → email delivered, ticket status resolved.

| Risk | Blast Radius | Mitigation |
|------|-------------|------------|
| Auth fallback breaks v2 Supabase Auth users | Medium — only affects if real Supabase Auth users exist | Guard: fallback only runs when `get_user_context` returns None |
| Email threading appends to wrong ticket | Low — only when gmail_thread_id matches | Use exact match; log when match found |
| Dashboard redirect loops | Low — if brands API 401s, navigate() called in loop | Wrap in try/catch; only redirect on explicit `brands.length === 0` |
| Worker inner-loop change breaks existing poll | Low — purely additive try/except | No logic changed; only error isolation added |
